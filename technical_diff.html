Building a Blog

I consider myself to be pretty computer-savy, and you'd expect that putting up a blog would be a quick little task. Choose a service, pay some money, make a few cosmetic decisions and start writing; sounds pretty easy.

Well it wasn't like that at all.

I've been playing around with code and with Linux for a while, and I want to be able to build a project when I have an idea. (there's a cool paradox here) So my first thought was to use an old computer to set up a simple ubuntu server to goof around on, and to outsource things from my crazy laptop. I quickly ran out of time to do that, so in Rochester I got us a student discount for Digital Ocean's amazing and simple VPS service, which amounts to the same thing, but far easier.

The trouble is, I wanted to make a blog, instead of going to someone else to make it for me. (A service like squarespace or wix is for designing a blog, using pieces someone else made. It's a whole different game, I'm just not that into the design part.) But I still wanted things to be simple and easy, so I figured a blog-making platform like Wordpress would be a good fit; filled with control and customizability (being literally just a bunch of open source code) but still with navicable faculties for attractive internet publishing.

I put a blog-making package called Ghost (a simpler, newer version of Wordpress.org), only to find myself lost run into confusing, undocumented depths of DNS services, mysterious, rarely-used student programsbut was quickly feeling out of my depth. Finally, I decided that the best ballance between my desire to build it and the need to have a regular communication platform, was to use a very simple service offered by github.com called pages. It hosts html and css code (the bare bones of webpages, without fancy interactivity and calculation) easily and reliably. So here we are.

Thank you!
